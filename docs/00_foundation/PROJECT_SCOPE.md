# PROJECT SCOPE

## Current phase
This repository is currently in the theory and product-architecture phase for a personal-mode Communication DNA product.
The focus is not implementation yet, but preservation of context, mapping of hypotheses, exploration of product forms, and preparation for later MVP decisions.

## Core idea
Communication DNA is considered here as a system that analyzes human text communication and reveals:
- likely hidden needs;
- behavioral motives;
- communication structure;
- loss-of-contact points;
- distortions in interpretation;
- probable next useful move.

## Initial domain boundary
This repository currently focuses on **personal mode**.
Personal mode does not mean only romantic relationships. It includes any communication where a user is personally entangled in uncertainty, tension, hope, fear, attachment, projection, or identity stakes.

## Included topics
- one-to-one message analysis;
- analysis of the user’s own drafts and internal monologue;
- explicit and hidden user jobs;
- product formats;
- user effects and second-order effects;
- ethical boundaries and misuse risks;
- research questions and validation assumptions;
- documentation structures that preserve evolving theory without premature commitment.

## Excluded for now
- final product decisions;
- final UI choice;
- pricing and monetization lock-in;
- technical implementation architecture;
- training pipeline decisions;
- enterprise/work mode as a primary scope;
- group chat mode as a primary scope.

## Repository role
The repository is intended to preserve not only conclusions, but also:
- unresolved tensions;
- competing hypotheses;
- possible moves;
- rejected directions;
- reasoning artifacts worth returning to later.

This should prevent premature simplification and preserve the full theoretical field around the product.
