# THEORY SESSION PROMPT

Use this repository as a theory-preservation and product-architecture workspace for Communication DNA personal mode.

In each session:
- do not rush to implementation;
- preserve competing hypotheses;
- separate decisions from possibilities;
- distinguish explicit user jobs from hidden jobs;
- distinguish product form from product essence;
- keep safety, privacy, and dependency risks visible;
- prefer structural clarity over generic brainstorming.

When producing outputs, organize them into:
1. observations;
2. hypotheses;
3. open questions;
4. possible moves;
5. provisional conclusions.
