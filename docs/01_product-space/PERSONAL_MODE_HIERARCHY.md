# PERSONAL MODE HIERARCHY

## Purpose
This document formalizes personal mode not as a flat feature list, but as a layered system.

## Layer 1. Entry layer
This is how the user experiences the product.
Typical entry questions:
- what is happening here;
- what should I do next;
- am I overthinking this;
- what do I actually want;
- why did this trigger me;
- is this a repeated pattern;
- help me analyze my draft.

## Layer 2. Diagnostic layer
This is what the system must determine internally.

### Diagnostic axis 1. Material type
- incoming message;
- dialogue fragment;
- message history;
- user draft;
- internal monologue;
- relationship description.

### Diagnostic axis 2. Task type
- interpretation;
- next-move choice;
- projection check;
- desire clarification;
- trigger analysis;
- pattern analysis;
- closure / cycle completion.

### Diagnostic axis 3. User state
- calm;
- tense;
- anxious;
- emotionally captured;
- offended;
- frozen;
- impulsive.

### Diagnostic axis 4. Context depth
- one message only;
- short fragment;
- full conversation slice;
- relationship history;
- repeated-session user history.

### Diagnostic axis 5. Reading risk
- idealization;
- catastrophizing;
- wishful reading;
- confirmation seeking;
- dependency on external permission;
- manipulative intent.

## Layer 3. Transformational layer
This is the long-term effect the product may create:
- lower projection load;
- stronger reality-based reading;
- more mature response selection;
- more tolerance for ambiguity;
- stronger sense of position;
- better pattern recognition across relationships;
- more capacity to close destructive cycles.

## System interpretation
A strong personal-mode product should work across all three layers:
- user entry;
- internal diagnosis;
- long-term transformation.
