# USER SCENARIOS

## Scenario classes

### 1. Interpret ambiguous incoming message
The user wants to understand whether the message indicates interest, avoidance, control, guilt, hostility, or simple neutrality.

### 2. Decide whether to answer
The user is stuck between responding, waiting, confronting, or withdrawing.

### 3. Draft response before sending
The user wants to test tone, likely impact, hidden motive, and whether the reply supports their real aim.

### 4. Post-conflict review
The user wants to understand what actually happened and where the interaction broke.

### 5. Pattern review across multiple dialogs
The user suspects repeated life scripts, not a one-off misunderstanding.

### 6. Internal monologue analysis
The user inputs notes, unsent messages, or self-talk to detect hidden need, distortion, or false goal.

## Scenario design note
The product should handle both acute moments and reflective review. These are different emotional states and may require different output structures.
