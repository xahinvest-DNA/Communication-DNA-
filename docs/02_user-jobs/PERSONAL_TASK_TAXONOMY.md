# PERSONAL TASK TAXONOMY

## Purpose
This document maps personal mode by user task types rather than by relationship category.

## Core task classes

### 1. Interpretation
The user wants to know what a message, exchange, or silence likely means.

### 2. Next move selection
The user wants to decide whether to reply, clarify, pause, confront, withdraw, or set a boundary.

### 3. Projection check
The user wants to know whether they are overreading, idealizing, catastrophizing, or seeking certainty rather than truth.

### 4. Desire clarification
The user wants to understand what they themselves actually want: contact, control, reassurance, self-respect, closure, validation, or something else.

### 5. Trigger / capture recovery
The user is emotionally hooked and needs distance, stabilization, and restoration of position before making a move.

### 6. Dynamic / pattern analysis
The user wants to know whether this is a one-off event or a repeated relational pattern.

### 7. Closure / cycle completion
The user wants to stop feeding a destructive loop, exit with dignity, or complete an unfinished inner cycle.

### 8. Self-analysis through text
The user wants to analyze drafts, notes, inner monologue, or unsent messages to surface hidden motives, false goals, or internal conflict.

## Meta-tasks under the surface
Across these task classes, users often seek one or more hidden outcomes:
- clarity;
- permission;
- relief from uncertainty pain;
- restoration of inner control;
- preservation of self-respect / self-coherence.

## Strategic implication
Product design should not be organized primarily around romance, friendship, family, or work labels. It should be organized around recurring task structures that can appear across all those domains.
